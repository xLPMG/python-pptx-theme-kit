You are an LLM tasked with creating a PowerPoint presentation script in this repository.

Your output should be Python code that uses the local toolkit in src/python_pptx_theme_kit, not raw ad-hoc python-pptx calls for everything.

Use these imports:

from python_pptx_theme_kit import (
		Presentation,
		Inches,
		PP_ALIGN,
		get_palette,
		list_palettes,
		make_blocks,
		make_primitives,
		detect_overlaps,
		format_overlaps,
)

Repository features you can and should use

1. Theme palettes
- get_palette(name) gives a consistent color system.
- Available palette names:
	- catppuccin_latte
	- catppuccin_frappe
	- catppuccin_macchiato
	- catppuccin_mocha
	- nord_frost
	- paper_sunrise

2. Blocks builder (preferred)
- b = make_blocks(palette)
- Prefer blocks first, then primitives only for gaps/overrides.
- b is a dict with these helpers:
	- b["slide_chrome_block"](slide, title=None, subtitle="", footer_text=None, show_title_bar=True, bg_color=None)
	- b["hero_banner_block"](slide, title, subtitle, org_line, objective, band_top=Inches(2.3), band_height=Inches(2.9))
	- b["toc_list_block"](slide, items, left=Inches(0.35), top=Inches(1.15), width=Inches(12.6), row_height=Inches(0.52), row_gap=Inches(0.05), number_width=Inches(0.52), page_width=Inches(1.9), section_size=13, page_size=12)
	- b["data_source_cards_block"](slide, sources, left=Inches(0.35), top=Inches(1.15), width=Inches(12.6), height=Inches(3.85), columns=3, gap=Inches(0.2), row_height=Inches(0.33))
	- b["coverage_table_block"](slide, column_headers, rows, left=Inches(0.35), top=Inches(1.15), width=Inches(12.6), header_height=Inches(0.4), row_height=Inches(0.43), first_col_width=Inches(4.0), insight_text=None, status_positive="Y", status_negative="-")
	- b["pipeline_stages_block"](slide, stages, left=Inches(0.4), top=Inches(2.0), width=Inches(12.5), box_height=Inches(1.3), connector_text=">")
	- b["pipeline_strategy_block"](slide, heading, strategy_text, code_text, top=Inches(3.55), width=Inches(12.6), left=Inches(0.35))
	- b["code_status_block"](slide, code_text, status_rows, code_heading="Implementation", status_heading="Status Snapshot", left=Inches(0.6))
	- b["image_caption_card_block"](slide, image_path, title, caption, left=Inches(0.35), top=Inches(1.15), width=Inches(12.6), height=Inches(5.9), image_ratio=0.78, fit="cover")
	- b["two_column_panel_block"](slide, left_title, left_items, right_title, right_items, left=Inches(0.6), top=Inches(2.0), width=Inches(12.13), height=Inches(4.4), gap=Inches(0.23))
	- b["left_border_card_list_block"](slide, items, left=Inches(0.35), top=Inches(1.15), width=Inches(12.6), row_height=Inches(1.0), row_gap=Inches(0.08), stripe_width=Inches(0.1), title_width=Inches(2.2), title_size=12, desc_size=11, alternating_bg=False)

3. Primitive builder (fallback/custom)
- p = make_primitives(palette)
- p is a dict with these helpers:
	- p["blank_slide"](prs)
	- p["set_bg"](slide, color)
	- p["add_rect"](slide, left, top, width, height, fill_color, line_color=None, line_width=None)
	- p["add_text"](slide, text, left, top, width, height, size=14, bold=False, italic=False, color=..., align=PP_ALIGN.LEFT, wrap=True)
	- p["add_image"](slide, path, left, top, width=None, height=None, fit="contain", align="center", border_color=None, border_width=None)
	- p["add_code"](slide, code, left, top, width, height, size=10, min_size=8, wrap=False, truncate=True)
	- p["title_bar"](slide, title, subtitle="", bar_height=Inches(1.05))
	- p["section_label"](slide, text, left, top, width, height=Inches(0.3), size=13, color=..., align=PP_ALIGN.LEFT)
	- p["bullet_block"](slide, items, left, top, width, height, size=13, color=..., bullet="•")
	- p["info_row"](slide, label, value, y, row_h=Inches(0.52), lw=Inches(2.8), label_color=..., value_color=...)
	- p["footer"](slide, text)

4. Overlap quality check
- detect_overlaps(prs, min_overlap_ratio=0.01, ignore_full_containment=True, containment_ratio=0.98)
- format_overlaps(findings)
- Use this at the end of generation and print warnings if overlaps are found.

Image fit mode guidance

- `fit="contain"`: keep full image visible inside frame (no crop)
- `fit="cover"`: fill frame completely, crop overflow
- `fit="stretch"`: force exact frame size (aspect ratio may distort)
- `fit="native"`: keep native proportions when one dimension is omitted

Required slide setup

- Create a Presentation() and set:
	- prs.slide_width = Inches(13.33)
	- prs.slide_height = Inches(7.5)

Design and layout rules

- Create a multi-slide deck (at least 5 slides unless user requests fewer).
- Maintain visual consistency with one selected palette.
- Use b["slide_chrome_block"] on most slides for coherent background/title/footer.
- Prefer block composition for known patterns before using primitives.
- Use section_label for scoped regions (columns/cards), not full-width assumptions.

- Mix prepared blocks across deck when possible:
	- TOC list
	- comparison table
	- left-border card list
	- process pipeline
	- code+status section
	- image card sections

- Use primitives for custom detail layers:
	- narrative text blocks
	- bullet lists
	- code blocks
	- key-value rows
	- decorative rectangles/cards
- Keep readable spacing and avoid crowding.
- Prefer p["add_code"] with wrap=True for long lines.
- Do not place multiple large text boxes on top of each other.
- Avoid ultra-short text boxes for captions/labels; leave enough height so text baselines render clearly inside the box.

Output requirements

- Provide complete runnable Python code, including:
	- imports
	- main() function
	- slide construction
	- use of make_blocks as the primary composition API
	- save call
	- overlap check output
	- if __name__ == "__main__": main()
- Save to a descriptive file name, for example:
	- generated_deck.pptx
- Print the saved filename.
- After saving, run overlap detection and print either:
	- "No significant overlaps detected."
	- or a bullet list of overlap lines from format_overlaps.

Style goals

- Build slides that look intentional and modern.
- Use hierarchy: title, subtitle/context, body, footer.
- Balance dense slides with simpler visual slides.
- Ensure generated content feels polished and presentation-ready.
